[CRITICAL] FC saat scan QR — Base64 mismatch APK vs Admin Panel
Admin Panel v2 menghasilkan Base64 standar yang mengandung karakter +, /, dan =. APK men-decode dengan mode yang tidak bisa membaca karakter tersebut, menyebabkan payload AES corrupt dan app langsung crash. Keduanya sekarang sinkron menggunakan URL-safe Base64.
Base64.DEFAULT → Base64.URL_SAFE or Base64.NO_PADDING
ScanActivity.kt
🔴
[CRITICAL] FC setelah scan QR di Android 14
Pemanggilan startForeground() 2-argumen menyebabkan crash di Android 14 (API 34) karena wajib deklarasi tipe service secara eksplisit. Fix: migrasi ke varian 3-argumen.
startForeground(id, notif) → startForeground(id, notif, FOREGROUND_SERVICE_TYPE_DATA_SYNC)
SecurityService.kt
🟠
[HIGH] SecurityService berhenti setelah scan berhasil
Race condition antara onDestroy ScanActivity dan startService menyebabkan service tidak jalan stabil setelah navigasi ke ExamActivity. Service kini di-start ulang di onCreate ExamActivity.

⌨️
Anti-Custom Keyboard — Blokir keyboard pihak ketiga
Deteksi apakah keyboard yang aktif bukan keyboard bawaan perangkat. Menggunakan Settings.Secure.DEFAULT_INPUT_METHOD dan whitelist package keyboard sistem (Samsung, MIUI, AOSP, Gboard, ColorOS, dll). Jika custom keyboard terdeteksi, ujian tidak bisa dimulai.
checkCustomKeyboard() → blocked if currentIme ∉ allowedKeyboards
SecurityGuard.kt ✨ ScanActivity.kt ExamActivity.kt
📋
Clipboard Wiper — Auto-wipe di onResume & onWindowFocusChanged
ClipboardGuard otomatis menghapus isi clipboard (ClipboardManager.clearPrimaryClip() di API 28+, fallback setText("") untuk API lama) saat Activity kembali ke foreground atau mendapat fokus. Continuous guard listener juga wipe setiap kali ada data baru masuk clipboard selama ujian berlangsung.
lifecycle.addObserver(clipboardGuard) + addPrimaryClipChangedListener
ClipboardGuard.kt ✨ ScanActivity.kt ExamActivity.kt
♿
Accessibility Service Blocker — Blokir app pembaca layar berbahaya
Malicious app pakai Accessibility Service untuk baca teks layar (bypass FLAG_SECURE) atau simulasi tap. Implementasi cek via AccessibilityManager.getEnabledAccessibilityServiceList(FEEDBACK_ALL_MASK). Jika ada non-system accessibility service aktif, ujian ditolak dengan pesan detail nama service.
am.getEnabledAccessibilityServiceList() → filter non-system packages
SecurityGuard.kt ✨ ScanActivity.kt
🖥️
Secondary Display & OTG Monitor — Blokir screencast real-time
DisplayGuard menggunakan DisplayManager.DisplayListener untuk mendeteksi display sekunder (HDMI, Miracast, Chromecast, virtual display). Cek awal saat onResume (displayId != 0), monitoring real-time selama ujian. Jika monitor eksternal atau screencast aktif saat ujian: violation dicatat langsung.
displayManager.displays.any { it.displayId != 0 } → lock exam
DisplayGuard.kt ✨ ExamActivity.kt
🔐
VPN & Proxy Detection — Enhanced, cek real-time saat ujian
Deteksi VPN via NetworkCapabilities.hasTransport(TRANSPORT_VPN). Ditambah deteksi HTTP proxy manual via System.getProperty("http.proxyHost"). Re-check dilakukan saat onResume ExamActivity — jika user nyalain VPN setelah layar dibuka, tetap ter-detect dan dicatat sebagai violation. Debounce 30 detik masih aktif dari SecurityService.
TRANSPORT_VPN || http.proxyHost != null → violation "vpn_active"
SecurityGuard.kt ✨ ExamActivity.kt
📌
SSL Certificate Pinning — Blokir MitM attack (HTTP Toolkit, Charles)
ExamCertificatePinner menggunakan OkHttp CertificatePinner untuk mengikat koneksi ke SHA-256 fingerprint certificate server ujian. Jika seseorang menginstal CA palsu (Burp, Charles, HTTP Toolkit) untuk intercept traffic, koneksi langsung gagal dengan SSLPeerUnverifiedException. Mendukung primary + backup pin untuk antisipasi rotasi certificate.
CertificatePinner.Builder().add(domain, "sha256/...").build()
ExamCertificatePinner.kt ✨ network_security_config.xml ✨
🛡️
Pre-exam Security Gate — Semua cek dijalankan sebelum tombol aktif
SecurityGuard.runAllChecks() menjalankan semua cek baru (keyboard, accessibility, display, VPN/proxy) sebagai satu pipeline. Dipanggil di onResume ScanActivity dan sesaat sebelum ExamActivity dibuka. Jika gagal, dialog blocking muncul dengan kode error dan instruksi perbaikan.
val r = guard.runAllChecks(); if (!r.passed) showBlockDialog(r.reason, r.code)
SecurityGuard.kt ✨ ScanActivity.kt
📲
AndroidManifest.xml — Permission baru untuk security features
Ditambahkan CHANGE_NETWORK_STATE (backup VPN detection) dan network_security_config untuk domain pinning. Tidak ada permission yang memerlukan user approval di runtime.
AndroidManifest.xml res/xml/network_security_config.xml ✨

[CRITICAL] App tidak muncul di drawer Samsung setelah install
Penyebab: SplashActivity memiliki intent-filter category HOME dan DEFAULT. Pada Samsung (OneUI), ini menyebabkan launcher chooser dialog muncul dan setelah dibatalkan app tidak tampil di app drawer. Hanya LAUNCHER category yang diperlukan untuk MAIN activity.

AndroidManifest.xml
🔴
[CRITICAL] Compile error: tvActionHint type mismatch
Layout mendeklarasikan tvActionHint sebagai LinearLayout, tapi kode Kotlin memanggil .text dan .setTextColor (method milik TextView). Ini menyebabkan compile error dan app tidak bisa di-build sama sekali. Fix: ID dipindahkan ke inner TextView di dalam LinearLayout.
android:id="@+id/tvActionHint" → pindah ke <TextView> bukan <LinearLayout>
activity_scan.xml
🟠
[HIGH] hasUsageStatsPermission() selalu return true
Implementasi sebelumnya menggunakan queryUsageStats() dan cek apakah hasilnya null. Masalahnya: Android return empty list (bukan null) ketika izin tidak diberikan. Akibatnya, tombol ujian bisa diaktifkan tanpa izin monitor. Fix: Gunakan AppOpsManager.checkOpNoThrow() yang merupakan cara yang benar untuk cek izin ini.
AppOpsManager.MODE_ALLOWED → izin benar-benar diberikan
SecurityManager.kt
🟠
[HIGH] Notification icon tampil kotak aneh di status bar
R.mipmap.ic_launcher digunakan sebagai small notification icon. Android mengharuskan small icon berupa alpha-only monochrome drawable. Icon berwarna akan ditampilkan sebagai kotak putih solid atau blob abu-abu di status bar. Fix: Buat ic_notification.xml sebagai vector drawable monochrome.
setSmallIcon(R.drawable.ic_notification) ← vector monochrome
